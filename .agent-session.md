## Known Issues

### 🔴 Database Connection Broken
The `.env` `DATABASE_URL` is a Prisma Accelerate URL (`prisma+postgres://...`) but the generated Prisma client uses the adapter pattern (`@prisma/adapter-pg`). These are incompatible. **All database writes fail silently** (the vision page handles this gracefully — analysis works without persistence).

**To fix:** Either:
- Option A: Replace `.env` `DATABASE_URL` with a direct `postgresql://user:pass@host:5432/db` connection string
- Option B: Regenerate Prisma client for Accelerate mode
